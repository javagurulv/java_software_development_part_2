Ментор хочет чтобы вы внедрили в проект статический анализ кода.

Узнайте что это такое, для чего он нужен и как работает.

Ваша задача: добавьте в проект Gradle плагин id 'checkstyle'.

plugins {
	id 'org.springframework.boot' version "2.7.5"
	id 'com.palantir.docker' version '0.35.0'
	id 'checkstyle'
}

Так как конфигурация статического код анализатора будет лежать
у вас в модуле, добавьте следующие строки в ваш gradle.build:

checkstyle {
	configFile = 'config/checkstyle/checkstyle.xml' as File
}

Скопируйте папку config из этого шага себе в модуль.

Запустите билд вашего модуля и убедитесь, что статический код
анализ работает. В первый раз он выдаст очень много ошибок.
Не волнуйтесь, попробуйте исправить самые очевидные из них.

Если это необходимо (чтобы ошибки статического код анализа)
не фейлили билд укажите для них свойство severity -> warning:

        <module name="FinalClass">
            <property name="severity" value="warning"/>
        </module>

или

    <!-- Глобальное свойство - делает все проверки = warning!
    <property name="severity" value="warning"/>
    -->


Добейтесь успешного прохождения код анализатора и зеленого билда!
