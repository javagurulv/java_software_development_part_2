Запускаем два контейнера в одной сети!

В прошлом задании вы создали виртуальную сеть.
Теперь пришла пора запустить в этой сети два
контейнера: MySQL сервер и Insurance Calculator.

При работе с виртуальной сетью имена хостов = именам контейнеров.
То есть обратиться из одного контейнера в другой в рамках оной
виртуальной сети можно указав вместо IP адреса название контейнера.
Оба контейнера должны быть запущены в одной виртуальной сети.

Ваша глобальная задача для этого шага: поднять два контейнера
в одной виртуальной сети и подключить приложение к базе данных
развернутой в MySQL контейнере.

Шаг 1: Переключите конфиг приложения на MySQL сервер.
Используйте имя MqSQL контейнера вместо хоста.

Шаг 2: Проверьте создана ли виртуальная сеть (docker network ls),
если не создана то создайте.

Шаг 3: Запустите контейнер с MySQL сервером и базой данных
в виртуальной сети.

Шаг 4: Соберите контейнер с приложением и активной конфигурацией
для подключения к MySQL серверу и базе внутри контейнера.

Шаг 5: Запустите контейнер с приложением.

Шаг 6: Подключитесь к базе данных используя MySQL Workbench
и убедитесь, что в базе данных приложение при стартапе создало
все таблицы (это должен сделать Liquibase).
