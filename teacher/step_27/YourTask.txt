Учимся использовать Docker volumes.

По бизнес задаче нам надо сгенерировать pdf документ и сохранить его
для дальнейшего использования. Документы будем хранить на диске. Для
того чтобы сохранять документы на диске из приложения запущенного
внутри контейнера можно при запуске контейнера дать доступ к
директории из файловой системы нашей OS. Это можно сделать
при помощи фичи volumes в docker-compose.

Узнайте как работает фича volumes в Docker.

Создайте директорию в проекте doc-generator-app под названием
docs/proposals.

Путь до директории где будут храниться файлы предложений
должен быть конфигурируемый через параметры приложения.
Поэтому добавьте в конфигурацию приложения doc-generator-app
параметр:
proposals.directory.path=/app/docs/proposals

Добавьте секцию volumes в docker-compose файле для
приложения doc-generator-app:
    volumes:
      - ./doc-generator-app/docs/proposals:/app/docs/proposals

При обработке сообщения сохраните его просто в текстовый файл
в указанной директории просто для проверки функциональности.

Запустите контейнеры и проверьте, создается ли файл
в указанной директории.
