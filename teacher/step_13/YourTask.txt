Последовательность выполнения Gradle задач.

В прошлых заданиях мы добавили в проект Gradle docker задача
для создания Docker image для нашего приложения.

Если выполнить команду Gradle clean которая удаляет все Gradle
артефакты созданные им при сборке проекта в папке build (в том
числе и jar файл приложения), а потом сразу выполнить команду
Gradle docker, то она упадет так как в папке build не будет
jar файла приложения.

Ваша задача: сделайте так, чтобы Gradle docker задача зависела
от Gradle bootJar (то есть выполнялась после нее). Для этого
воспользуйтесь параметром dependsOn, который есть у каждой Gradle задачи.
Если вы все сделаете правильно, то перед выполнением Gradle docker
всегда будет выполняться задача Gradle bootJar, которая и будет
создавать jar файл, этот jar файл потом будет использовать
Gradle docker для создания Docker image.

В этом шаге мы узнали как с помощью параметра
dependsOn можно выстраивать зависимости между Gradle задачами.
