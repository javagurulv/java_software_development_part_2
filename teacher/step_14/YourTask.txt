Учимся использовать Spring Profiles!

В прошлом задании мы запустили приложение в контейнере
и подключили его к MySQL серверу запущенному в другом контейнере.

Для этого нам пришлось поменять параметры конфигурации datasource
в application.properties файле. Теперь они переключены на использование
MySQL базы данных.

А что если мы захотим запустить приложение локально с H2
базой данных или переключить приложение на использование
локально установленного MySQL сервера? Нам опять придется
менять application.properties конфигурацию?

Как то это выглядит не очень! А есть ли другой способ?

Да есть! Это Spring Profiles!

Прочитайте про эту функциональность.
Вот неплохая статья: https://www.baeldung.com/spring-profiles

Ваша задача: с помощью файлов конфигурации настроить три Spring профайла:
- для запуска приложения с H2 базой;
- для запуска приложения локально с MySQL локальной базой;
- для запуска приложения в контейнере с MySQL базой в другом контейнере;

Для выбора Spring профайла используйте environment variable:

-Dspring.profiles.active=h2
-Dspring.profiles.active=mysql-local
-Dspring.profiles.active=mysql-container

Поднимите приложение с разными профайлами
и проверьте что все работает корректно!
